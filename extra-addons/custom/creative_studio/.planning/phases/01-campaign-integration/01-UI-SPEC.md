## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (Odoo default) | 400 (regular) | 1.5 |
| Small / Label / Badge | 11px | 600 (semibold) | 1.3 |
| Heading | 20px (h2 Odoo default) | 600 (semibold) | 1.2 |

**Weight system:** 2 weights only -- 400 (regular) for body text and muted secondary text, 600 (semibold) for labels, headings, badges, and any text requiring emphasis.

**Consolidation note:** The previous 4-size scale (11px, 12px, 14px, 20px) had 11px and 12px only 1px apart. Consolidated to 3 sizes: 11px for all small/label/badge text, 14px for body, 20px for headings. The campaign badge, step badges, review badges, and active-step labels all use 11px/600. Status badges (`.proofing-status-approved` etc.) that previously used 13px/600 are outside this phase's scope and remain unchanged.

Source: Extracted from existing `proofing.css` `.proofing-step-badge` (12px/600 consolidated to 11px/600), `.proofing-review-badge` (11px/500 updated to 11px/600), `.proofing-status-approved` (13px/600 unchanged).

### Campaign Badge Specification

**Position:** Inline after the project name `<h2>` in the dashboard header, with `ms-3` (12px left margin in LTR, auto-flipped by rtlcss for RTL).

**Visual:**
```
[bullhorn-icon] Campaign Name
```

- Class: `badge proofing-campaign-badge ms-3`
- Background: `rgba(99, 102, 241, 0.15)` (matches `.proofing-step-badge`)
- Text color: `#818cf8`
- Border: `1px solid rgba(99, 102, 241, 0.25)`
- Border radius: `6px`
- Padding: `4px 12px`
- Font size: `11px`
- Font weight: `600`
- Cursor: `pointer`
- Icon: `fa fa-bullhorn me-1` (FontAwesome 4)
- Hover: `background: rgba(99, 102, 241, 0.25)` (slightly more opaque)

**Behavior:** Click triggers `this.action.doAction()` navigating to `utm.campaign` form view. Badge is read-only display (D-09). If no campaign is linked, badge is hidden entirely (no empty placeholder in dashboard header).

**RTL:** Uses `ms-3` (margin-inline-start) which rtlcss converts correctly. No `/*rtl:ignore*/` needed.

### Project Card in Campaign Form Specification

**Layout:** Kanban card rendered inside One2many field on the campaign form notebook tab.

**Card contents (per D-06):**
1. **File thumbnails** -- First 4 image files, displayed as 48x48px rounded squares in a horizontal row with 4px gap. Uses `/web/image/proofing.file/{id}/thumbnail` URLs. Non-image files show a file-type icon placeholder.
2. **Approval progress** -- Fraction text format: "3/7 approved" (not a progress bar). Uses a computed `approval_summary` Char field. Fraction text chosen over progress bar because it is more precise and matches the existing `decision_counts` pattern ("0/0" format) already in the codebase.
3. **Reviewer avatars** -- 28x28px rounded circles, max 5 shown, "+N" overflow badge if more. Uses existing `proofing-reviewer-avatar` pattern.
4. **Active step name** -- Small label: "Step: Design Review" in muted text (11px, weight 400).
5. **Latest version** -- Small label: "v3" in muted text (11px, weight 400).
6. **Project name** -- Card title, 14px weight 600, clickable to open project form. This is the primary visual anchor of each card.

**Card dimensions:** Min-width 280px, max-width 360px. Padding 16px. Background `var(--o-webclient-background-color)`. Border `1px solid var(--o-border-color)`. Border-radius 8px.

## Interaction Contracts

### Campaign Badge Click (Dashboard)
- **Trigger:** Click on campaign badge in dashboard header
- **Action:** Navigate to `utm.campaign` form view via `this.action.doAction({ type: 'ir.actions.act_window', res_model: 'utm.campaign', res_id: campaign_id, views: [[false, 'form']], target: 'current' })`
- **Feedback:** Standard Odoo breadcrumb navigation (user can click back)
- **Guard:** Badge only renders when `state.project.campaign_id` is truthy

### Campaign Field on Project Form
- **Trigger:** User clicks Many2one field dropdown on proofing project form
- **Action:** Standard Odoo Many2one widget behavior (search, select, quick-create)
- **Options:** `{'no_quick_create': True, 'create_name_field': 'title'}` -- prevents accidental campaign creation, uses `title` field for any create action
- **Unlink:** User clears the field (standard Many2one clear button). Field becomes `False`. Project disappears from campaign's tab on next refresh.

### Project Card Click (Campaign Form)
- **Trigger:** Click on project name in kanban card on campaign form tab
- **Action:** Standard Odoo kanban record open (navigates to proofing.project form)
- **Feedback:** Standard Odoo breadcrumb

| Element | English | Arabic |
|---------|---------|--------|
| Campaign badge label | `{campaign_title}` (dynamic) | `{campaign_title}` (dynamic, from utm.campaign.title) |
| Campaign badge tooltip | "View Campaign" | "عرض الحملة" |
| Form field label | "Campaign" | "الحملة" |
| Tab label on campaign form | "Proofing Projects" | "مشاريع المراجعة" |
| Empty state (campaign tab, no projects) | "No proofing projects linked to this campaign." | "لا توجد مشاريع مراجعة مرتبطة بهذه الحملة." |
| Empty state next step | "Link a project from its form view." | "اربط مشروعا من صفحة النموذج الخاص به." |
| Approval fraction | "{N}/{M} approved" | "{N}/{M} معتمد" |
| Active step label | "Step: {step_name}" | "المرحلة: {step_name}" |
| Version label | "v{N}" | "v{N}" |

**Translation pattern:** All new strings added to `_get_dashboard_translations()` method (English and Arabic sections). Campaign form XML strings use Odoo's standard XML `string` attribute for po-file extraction. No `_t()` calls in JS (broken for custom modules per project memory).

**Primary CTA:** None for this phase. Campaign linking is done via standard Many2one field on the project form (D-09), not a dedicated CTA button.

**Destructive actions:** None. Unlinking a campaign is simply clearing a Many2one field (D-11), which is a standard Odoo interaction with built-in undo via the discard button. No confirmation dialog needed.

**Error state:** No custom error states. Campaign field uses standard Odoo Many2one validation. If `utm.campaign` record is deleted while a project references it, `ondelete='set null'` handles it silently (D-10).

## State Diagram

```
Campaign Link States (proofing.project.campaign_id):

  [No Campaign] ---(user sets field on form)---> [Campaign Linked]
  [Campaign Linked] ---(user clears field on form)---> [No Campaign]
  [Campaign Linked] ---(campaign record deleted)---> [No Campaign]  (via ondelete='set null')

Dashboard Badge Visibility:
  campaign_id is truthy  --> badge visible, clickable
  campaign_id is falsy   --> badge hidden (no empty placeholder)
```

## Accessibility Notes

- Campaign badge uses `cursor: pointer` and responds to click -- must also be keyboard-accessible (Odoo's OWL handles this for `t-on-click` on native elements)
- Badge icon (`fa fa-bullhorn`) is decorative; the adjacent text provides the label
- Campaign badge tooltip ("View Campaign") provides additional context for screen readers via `title` attribute
- Color contrast: `#818cf8` on `rgba(99, 102, 241, 0.15)` background passes WCAG AA for the badge text against the semi-transparent background when rendered on typical Odoo light or dark surfaces
- Reviewer avatars on campaign cards include `title` attribute with reviewer name

## RTL Considerations

- Campaign badge uses `ms-3` (margin-inline-start) -- rtlcss auto-flips to `me-3` in RTL. Correct behavior.
- Badge icon uses `me-1` (margin-inline-end) -- rtlcss auto-flips. Correct behavior.
- Campaign card layout uses flexbox with `gap` -- direction-agnostic. No intervention needed.
- No absolute positioning or `translate()` transforms in this phase's CSS additions.
- All new strings support Arabic (see Copywriting Contract above).
