**3. In `project_dashboard.js` — Add `onCampaignClick()` method:**

Add a new method to the `ProofingDashboard` class. Place it near the other `on*` click handler methods:

```javascript
onCampaignClick() {
    if (this.state.project && this.state.project.campaign_id) {
        this.action.doAction({
            type: "ir.actions.act_window",
            res_model: "utm.campaign",
            res_id: this.state.project.campaign_id,
            views: [[false, "form"]],
            target: "current",
        });
    }
}
```

**4. In `project_dashboard.xml` — Add campaign badge in header:**

Inside the `proofing-header` div, immediately AFTER the closing `</h2>` tag and BEFORE the closing `</div>` of the header, insert:

```xml
<t t-if="state.project.campaign_id">
    <span class="badge proofing-campaign-badge ms-3"
          t-on-click="onCampaignClick"
          t-att-title="T.viewCampaign"
          role="button">
        <i class="fa fa-bullhorn me-1"/>
        <t t-esc="state.project.campaign_name"/>
    </span>
</t>
```

Note: `state.project.campaign_name` is populated from `get_dashboard_data()`. The `t-att-title` uses the server-side translation for the tooltip. The `role="button"` aids accessibility. Uses `ms-3` for inline-start margin (RTL-safe per UI-SPEC).
